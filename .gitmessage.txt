# Git 커밋 메시지 템플릿
# 
# 작성 규칙:
# 1. 제목(첫 줄)은 50자 이내로 간결하게 작성
# 2. 제목 끝에 마침표(.) 금지
# 3. 제목은 명령문 형태로 작성 (ex: '버그 수정' → '버그 수정함')
# 4. 본문은 필요할 때만 작성, 72자 단위 줄바꿈 권장
# 5. 이슈 번호가 있다면 커밋 제목 끝에 (#이슈번호) 형식으로 작성
# 6. 타입은 반드시 소문자 영어로 작성
#
# 커밋 타입:
# feat:     새로운 기능 추가
# fix:      버그 수정
# docs:     문서 수정
# style:    코드 스타일, 포맷팅, 세미콜론 누락 등 수정 (기능 변경 없음)
# refactor: 코드 리팩토링 (기능 변경 없음)
# test:     테스트 추가 또는 수정
# chore:    빌드 업무 수정, 패키지 매니저 설정 등 (코드 변경 없음)
# perf:     성능 개선
#
# 예시:
# feat: 로그인 페이지 UI 구현 (#12)
# fix: 회원가입 폼 validation 에러 수정
# docs: README 파일 설치 가이드 추가
# style: ESLint 규칙에 맞게 코드 포맷팅
# refactor: 사용자 인증 로직 함수 분리
# test: 로그인 기능 단위 테스트 추가
# chore: package.json 의존성 업데이트
# perf: 이미지 최적화로 페이지 로딩 속도 개선
